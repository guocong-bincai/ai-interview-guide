## 🟢 基础概念题

### Q1: LLM 的 Token 是什么？

**答案要点：**
- Token 是 LLM 处理文本的基本单位
- 英文：1 token ≈ 4 个字符 ≈ 0.75 个单词
- 中文：1 个汉字 ≈ 1-2 个 tokens
- 影响：计费、上下文限制、生成长度

---

### Q2: Temperature、Top-P、Top-K 是什么？

**答案要点：**
- Temperature 控制随机性（0=确定，1=随机）
- Top-P 只从累积概率>P 的词里采样
- Top-K 只从概率最高的 K 个词里采样
- 问答用 Temperature=0，创作用 0.7

---

### Q3: 什么是 Context Window？超出限制怎么办？

**答案要点：**
- LLM 一次能处理的最大 token 数
- GPT-4o: 128K, GPT-3.5: 16K
- 解决方案：截断、总结、滑动窗口、向量检索

## 🟡 RAG 系统题

### Q4: RAG 的基本流程是什么？

**答案要点：**
1. 文档加载 → 2. 切分 → 3. 向量化 → 4. 存储 → 5. 检索 → 6. 生成

---

### Q5: Chunk 大小怎么选择？

**答案要点：**
- 太小：检索精确但语义不完整
- 太大：语义完整但检索不精确
- 经验值：通用 500-1000 tokens，技术 256-512 tokens

---

### Q6: 什么是 Embedding？

**答案要点：**
- 把文本转成向量（一串数字）
- 语义相似的文本向量接近
- OpenAI text-embedding-3-small: 1536 维

---

### Q7: 为什么用向量数据库？

**答案要点：**
- 传统数据库检索慢 O(N)
- 向量库用 ANN 索引快 O(log N)
- 10 万条数据：MySQL 5000ms → Qdrant 10ms

---

### Q8: 什么是混合检索？

**答案要点：**
- 向量检索 + 关键词检索（BM25）
- 向量懂语义，关键词精确匹配
- 取长补短，提升召回率

---

### Q9: 什么是 Rerank？

**答案要点：**
- 对检索结果重新排序
- 先检索 50 个候选，再精选 top-5
- 用 Cross-Encoder 模型，精度提升 15-25%

---

### Q10: RAG vs Fine-tuning 怎么选？

**答案要点：**
- RAG：海量、动态知识，需要引用溯源
- Fine-tuning：风格迁移、固定格式
- 企业知识库首选 RAG

---

## 🟠 Agent 开发题

### Q11: 什么是 AI Agent？

**答案要点：**
- 能自主决策和行动的 AI
- 核心组件：LLM + Tools + Memory + Planning
- 不仅能说话，还能干活

---

### Q12: ReAct 模式是什么？

**答案要点：**
- Reasoning + Acting
- 思考 → 行动 → 观察 → 循环
- 适用于需要调用外部工具的场景

---

### Q13: Function Calling 的原理？

**答案要点：**
- 将非结构化自然语言转为结构化 JSON
- 定义工具 Schema，LLM 决定调用哪个
- 实现自然语言查询数据库/API

---

### Q14: 如何防止 Agent 死循环？

**答案要点：**
- 最大轮次限制（如 10 轮）
- 超时机制（如 60 秒）
- 工具调用去重
- 反思机制

---

## 🔴 工程架构题

### Q15: 如何降低 LLM 调用成本？

**答案要点：**
1. 缓存（相同 query 直接返回）
2. 模型路由（简单问题用小模型）
3. Prompt 压缩（LLMLingua 减少 40% token）
4. 批量处理
5. 流式输出

---

### Q16: 如何监控 AI 应用？

**答案要点：**
- 指标：延迟、token 消耗、错误率、成本
- 追踪：完整请求链路（Prompt → Response）
- 评估：RAGAS、人工抽检、用户反馈
- 告警：成本异常、错误率飙升

---

### Q17: 如何处理高并发？

**答案要点：**
- API Gateway 限流（令牌桶）
- 多 Key 轮询缓冲
- 异步队列
- 超时容错 + Fallback

---

### Q18: 如何减少幻觉？

**答案要点：**
1. RAG（基于检索内容）
2. Prompt 约束（"不要编造"）
3. 引用溯源
4. 温度调低（<0.3）
5. 人工审核
